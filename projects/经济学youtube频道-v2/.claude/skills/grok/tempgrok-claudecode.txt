Claude Code 是 Anthropic 推出的代理式（agentic）编码工具，并非单纯的代码补全或聊天助手。它能在你的项目层面工作：读取整个代码库、跨文件规划和编辑、运行测试、修复 CI 错误、提交代码，甚至 24/7 自主执行任务。用户只需用自然语言描述目标，它就会使用真实开发工具链（Git、终端、GitHub CLI 等）自主迭代，直到完成。⁠Anthropic

它支持多种平台：终端 CLI（核心）、VS Code 扩展、桌面 App（macOS/Windows）、Web 版，并与 Claude Cowork、Claude Design 等产品深度集成。企业版零配置部署，已被 Stripe、Ramp、Wiz 等公司大规模采用（例如 Stripe 让 1370 名工程师用它完成万行级迁移）。⁠Anthropic

4. 4 月质量问题事件（官方 postmortem）

4 月中旬不少开发者反馈“Claude Code 变笨了”（响应重复、忘上下文、工具选择奇怪）。Anthropic 4 月 23 日发布详细 postmortem：

三处独立变更叠加导致：
3 月 4 日：默认 reasoning effort 从 high 降为 medium（为降低延迟）。
3 月 26 日：缓存优化 bug（每轮清除思考历史，而非闲置 1 小时）。
4 月 16 日：系统 prompt 增加“保持简短”限制（≤25/100 词），伤害编码质量。
4 月 7-20 日逐步修复（v2.1.116），4 月 23 日重置所有订阅者使用限额。
未来改进：更多内部员工使用公版、加强 Code Review 工具、prompt 变更严格 eval 等。⁠Anthropic
